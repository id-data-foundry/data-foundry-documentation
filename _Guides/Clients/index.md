# Data Foundry clients

The client section of this documentation contains all information you need to connect your platform or programming language of choice to Data Foundry. This includes basic information how to access Data Foundry from languages like Python and Javascript or platforms like Unity and MAX. All How-to guides will give you a general idea of how a specific client can be set up. It is recommended to have already followed the Data Foundry tutorials to gain an understanding of how the platform works.

Data Foundry is designed to be the backend for your data-enabled design projects. To make it easy to connect your prototypes, scripts, and applications to Data Foundry, we provide a set of client libraries and guides for popular creative coding platforms.

### Platform guides

For the supported platforms, we have specific guides and examples:

*   **[Python]({% link _Guides/Clients/Python.md %})**: Use Data Foundry with standard Python scripts or notebooks. Ideal for data analysis scripts, Raspberry Pi prototypes, and backend logic.
*   **[Processing]({% link _Guides/Clients/Processing.md %})**: Integrate data into your Processing sketches. Great for visual sketches and creative coding.
*   **[JavaScript]({% link _Guides/Clients/Javascript.md %})**: Connect web applications and hosted websites. Connect web-based prototypes and visualizations.
*   **[Unity]({% link _Guides/Clients/Unity.md %})**: Log data from 3D environments and games. Integrate data collection into games and VR/AR experiences.
*   **[Max]({% link _Guides/Clients/Max.md %})**: Interaction for media and sound projects. Connect interactive audio and media installations.

Explore the sub-pages to find specific installation instructions and code examples for your platform of choice.
