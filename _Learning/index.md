# Learning Materials

This section is dedicated to explanation and understanding, providing the conceptual foundation you need to work effectively with Data Foundry. It explains the 'why' behind the platform's features and the principles of the research context it operates in, helping you build a clear mental model of how everything fits together.

- **[About Data Foundry]({% link _Learning/DataFoundry/index.md %})**: To understand Data Foundry, it is essential to first learn about its core philosophy and purpose. This section explains the platform's origins, the problems it aims to solve in design research, and its underlying technical architecture. It provides the foundational knowledge for why the platform is built the way it is.

- **[Projects]({% link _Learning/Projects/index.md %})**: This section explains the most fundamental organizational concept in Data Foundry: the Project. You will learn how a project serves as the central container for all your research activities, bundling your data, sources, and team members. Understanding this structure is key to managing your work from start to finish.

- **[Datasets]({% link _Learning/Datasets/index.md %})**: Data is stored in Datasets, but choosing the right type is crucial. This section provides a conceptual overview of the different dataset types available, from storing sensor streams to survey results. It helps you learn to identify the correct tool for your specific data collection needs.

- **[Local AI]({% link _Learning/LocalAI/index.md %})**: Data Foundry integrates powerful AI capabilities directly into the platform. This section introduces the concepts behind the available models, explaining the trade-offs between local and external AI and the potential use cases. It provides the understanding needed to leverage these advanced features in your research.

- **[Open Source and Science]({% link _Learning/OpenSourceScience/index.md %})**: Effective and ethical research goes beyond just using tools. This section covers the important principles of Open Science, FAIR data, and licensing. It provides the conceptual framework to ensure your work is responsible, reusable, and contributes to a collaborative scientific community.
