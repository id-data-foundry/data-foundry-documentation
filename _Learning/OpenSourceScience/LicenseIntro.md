# What is a license

Licenses are a legally binding way to determine what someone is allowed to do with your: Code, Photos, Files or Data. By putting a license on your files you can protect and enforce how your files are used. If you want to for example share a picture online, but do not want companies to use that picture for commercial purposes, adding an appropriate license to the picture will help you protect this image. Sometimes you will be able to add a license to the file itself like you can do to pictures using Photoshop or Lightroom. For github repositories they usually add a single license file explaining the implications. Licenses can also be used to keep files or code open-source, these include sections on how the license also applies to derived work. To give you a general idea of the licenses out there, we just explain the most common licenses that can also be used on Data Foundry projects and datasets.

{: .warning}
This article gives a general explanation of what licenses mean and give you a general idea on how you can use licenses to protect your intellectual property, however these are not legal definitions so dont take the definitions as legal advice!

## Software Licenses

**The following licenses only apply to software (which means written code).**

### MIT license (default)

The MIT license is one of the most permissive licenses and essentially states that anyone can do whatever with the code or software sas long as you respect the copyright of the copyright holders and give them attribution for their work. Additionally every copy of a file that falls under the MIT license shall include this license or a reference to it. At last this license makes sure that the author of the file is not liable for any harm caused by files under the MIT license. Users can still infringe on patents by using patented MIT licensed code as long as the patent was granted before they started using the code. [More info can be found here.](https://choosealicense.com/licenses/mit/)

### Apache license

Similarly The Apache License 2.0 is a permissive open-source license that allows anyone to freely use, modify, and distribute the code, even in closed sourced software, as long as they comply with the license terms. It requires preservation of the copyright notice, the license text, and a NOTICE file (if present). In addition, if the patent-holder releases patented code under the apache license, users are legally allowed to use this code however they want as long as they comply to the rest of the license. [More info can be found here.](https://choosealicense.com/licenses/apache-2.0/)

### BSD-3-clause

Another MIT like license is the 3-Clause BSD license. This license is almost the same as the MIT license but adds one additional clause to make sure the creators or copyright holders identity can't be used to endorse the software without permission. This means that you can freely use the software, as long as you attribute the original copyright holders, but you cant use the copyright holders identity as any way of promotion of the project unless given permission to do so. 

### GPL-3.0 (Copyleft)

Whereas the other 3 licenses are more permissive, allowing users to do almost everything with your software, the GPL-3 license is more restrictive. This license tries to preserve the freedom of software and its derivatives. What this essentially means is that all projects using code licensed under GPL-3 should also be licensed under the GPL-3 license. This means that derivative works shall stay Open Source and provide the source code. Additionally if you patented your GPL-3 software, all users are freely able to use the code (as long as they follow the license). The GPL-3 license is often used to protect open-source code from being used in proprietary (locked down) software and hardware. The GPL-3 license allows companies to use this software commercially, and earn money, as long as they keep their own source-code open-source and freely accessible. [More info can be found here.](https://choosealicense.com/licenses/gpl-3.0/)

## Data and Creative work licenses

**The following licenses apply to all datasets and creative works you own the copyright to yourself. Code and software is excluded.**

### Creative Commons

The Creative Commons license is similar to the MIT license but focuses on creative work and data rather than software. By default it requires users of the creative work to attribute the copyright holder. What makes the Creative Commons license special is that it has several different variants, allowing you to specify what you want the files to be used for or not. Below we explain all the different variants. [more information can be found here.](https://creativecommons.org/share-your-work/cclicenses/)

#### Attribution [CC BY] (default)

The default variant to the CC (Creative Commons) license allows you to do almost everything with the data as long as you credit the creator when sharing this data. This also means that companies can use your creative work as long as they credit you.

#### Share-Alike [CC BY-SA]

The Share-Alike option of the CC license builds forward on the default variant, but also adds a copyleft clause. This means that all work built forward from your creative work modifying it, remixing or adapting it, should also remain under the same license. This prevents individuals from adapting your creative work and re-licensing it under a more restrictive license. This makes sure that all derived works remain open source, keeping software freely accessible throughout the whole software chain.

#### No-Derives [CC BY-ND]

The no-derives option allows you to prevent other users from changing your creative work. This means that they can not modify the original work and can only copy or share the original work, as long as credit is give to the creator. This is incompatible with the Share-Alike option as works under the No-Derives can't be derived, so no derivations can be shared.

#### Non-Commercial [CC BY-NC]

The Non-Commercial option to the CC license prevents companies or individuals from selling your work. This means that people are allowed to modify, share and copy the original work, as long as they do not make money in the process and proper credit is given. This makes it so that non-commercial individuals or institutions can use this work freely. This can be combined with both the Share-Alike clause (CC BY-NC-SA) or the No-Derives clause (CC BY-NC-ND).
